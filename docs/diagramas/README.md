# Diagramas de ZelogiG

Los diagramas se escriben en Mermaid y se versionan junto con el comportamiento que describen.

## Obligatorios

- `casos-de-uso.md`: siempre.
- `flujo-motor.md`: porque existe logica secuencial no trivial.
- `modelo-er.md`: porque existen tablas propias.
- `modelo-logico.md`: cuando cambien columnas, claves o relaciones.

## No obligatorio

BPMN no aplica por ahora: ZelogiG no es un proceso empresarial ni una automatizacion multi-actor.

## Regla

Un cambio de comportamiento o datos actualiza su diagrama en la misma tarea.
