# Reglas de colaboracion con IA

1. Leer PROJECT_GENESIS.md antes de proponer cambios.
2. Leer el documento del modulo que se modificara.
3. No inventar requisitos ausentes.
4. No cambiar el stack sin RFC.
5. No agregar dependencias por comodidad.
6. Mantener archivos pequenos y responsabilidades claras.
7. Separar dominio, aplicacion, interfaz e infraestructura.
8. Incluir pruebas para reglas del motor.
9. Registrar decisiones que afecten el futuro.
10. Explicar contradicciones antes de actuar.
11. No exponer secretos ni datos personales.
12. Entregar cambios pequenos y reversibles.
13. Actualizar documentacion cuando cambie el comportamiento.
14. No usar memoria conversacional como unica fuente de verdad.
