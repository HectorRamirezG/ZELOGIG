# Decisiones pendientes

Las siguientes decisiones no autorizan cambios. Deben presentarse al propietario con opciones, costo, riesgo, impacto y reversibilidad.

## D-001 Gestión de estado

- Estado: pendiente de evidencia.
- Base actual: reducer o controlador React sobre dominio puro.
- Alternativa futura: Zustand.
- Disparador: múltiples consumidores independientes, persistencia compleja o sincronización.

## D-002 Recursos visuales finales

- Estado: CSS y SVG aprobados para MVP.
- Pendiente: herramienta de IA para conceptos y condiciones de uso comercial.

## D-003 Sonido y música

- Estado: pendiente.
- Preguntas: identidad sonora, accesibilidad, licencias y control de volumen.

## D-004 Editor de niveles

- Estado: pospuesto hasta completar cinco niveles manuales.

## D-005 Supabase

- Estado: pospuesto hasta validar el juego local.

## D-006 Monetización

- Estado: exploración.
- Alternativas: demo más compra única, expansiones, edición educativa.
- Prohibido diseñar monetización que deteriore el puzzle o explote frustración.

## D-007 Graphify

- Estado: pendiente.
- Requiere evaluación de privacidad, costo operativo y utilidad real.

## D-008 Flujo Git

- Estado: repositorio inicial en `main`.
- Pendiente: decidir entre flujo simple por ramas cortas o `main/qa/dev` cuando exista un ciclo real de pruebas y publicación.
